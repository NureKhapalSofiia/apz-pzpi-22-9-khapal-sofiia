Короткий опис системи за минулою версією Vision & Scope:
Система призначена для покращення процесу догляду за шкірою в
медичних установах. Вона поєднує IoT-пристрій із сенсорами, що
вимірюють тип шкіри, рівень зволоженості та температуру, мобільний
додаток для лікарів і пацієнтів, веб-інтерфейс для адміністраторів та
централізовану серверну частину.
Після проведення аналізу дані автоматично передаються в систему,
де лікар має доступ до історії пацієнта, може аналізувати динаміку змін,
призначати засоби догляду та залишати рекомендації. Пацієнт отримує
персоналізовані поради, фіксує власні враження від засобів та може
комунікувати зі своїм лікарем.
Нижче наведено доповнені пункти документу:
3.1 Scope of Initial Release
Початкова версія програмної системи включає повноцінну
інтеграцію чотирьох основних компонентів: IoT-пристрою, мобільного
додатку, веб-клієнта для адміністраторів та серверної частини. Метою
релізу є забезпечення стабільної роботи всіх підсистем у рамках клінічного
середовища, з дотриманням стандартів безпеки та зручності для кінцевих
користувачів — лікарів, пацієнтів та адміністраторів.
IoT-пристрій
- Оснащений сенсорами для збору фізіологічних даних: тип шкіри,
рівень зволоженості, температура.
- Після завершення аналізу дані автоматично передаються на сервер
через бездротове з’єднання (Wi-Fi).
- Пристрій виконує локальну валідацію та базову обробку отриманих
даних.
Серверна частина
- Сервер обробляє всі запити від клієнтів, забезпечуючи логіку
доступу, зберігання та обробки даних.
- Реалізовані REST API для комунікації між IoT, мобільним додатком і
веб-інтерфейсом.
- Зберігання медичних даних відбувається в базі даних, з підтримкою
шифрування.
- Впроваджено систему аутентифікації та авторизації з розподіленням
прав доступу для лікарів, пацієнтів та адміністраторів.
Мобільний додаток
- Платформа: Android
Пацієнти можуть:
- Переглядати результати аналізів, історію змін шкіри.
- Отримувати персоналізовані рекомендації від лікаря.
- Вести власні нотатки щодо використаних засобів.
- Створювати чек-листи.
- Надсиласти повідомлення у чаті з лікарем
Лікарі можуть:
- Переглядати та аналізувати дані пацієнтів.
- Призначати доглядові засоби та планувати зустрічі.
- Вести електронні картки пацієнтів та переглядати динаміку.
- Надсиласти повідомлення у чаті з пацієнтом.
Веб-клієнт для адміністраторів
- Доступ через браузер без необхідності встановлення ПЗ.
Функції:
- Призначення лікарів пацієнтам.
- Керування базою сертифікованих доглядових засобів.
- Перегляд анонімізованої статистики користувачів та рейтингів
лікарів.
Взаємодія компонентів
- Всі частини системи синхронізовані через централізований сервер.
- Дані, зібрані пристроєм, обробляються на сервері й відображаються
у мобільному застосунку та веб-інтерфейсі.
- Кожен користувач має роль і рівень доступу, що обмежує або
розширює можливості взаємодії з даними.
- Комунікація між користувачами відбувається через внутрішні
повідомлення.
3.2 Scope of Subsequent Releases
У подальших версіях розроблюваної системи передбачається
інтеграція компонентів штучного інтелекту з метою підвищення точності
аналізу, автоматизації персональних рекомендацій та оптимізації взаємодії
між лікарями та пацієнтами. Нижче описані три основні аспекти роботи з
ШІ в межах цієї системи:
1. Застосування ШІ
Інтеграція ШІ дозволить значно розширити функціональність
системи:
- Автоматичне створення рекомендацій щодо засобів для догляду за
шкірою на основі аналізу поточних та історичних даних пацієнта (тип
шкіри, зволоженість, температура, реакції на попередні засоби).
- Прогнозування змін стану шкіри на основі часових рядів. Наприклад,
імовірність пересушення шкіри або появи подразнень при збереженні
поточного догляду.
- Виявлення аномалій або ризиків, які потребують втручання лікаря —
наприклад, нетипова зміна температури чи зволоженості, що може
свідчити про запалення.
2. Збирання даних
Для навчання і вдосконалення моделей ШІ система повинна
акумулювати наступні дані:
- Фізіологічні параметри шкіри з IoT-пристрою (тип, зволоженість,
температура) — з фіксацією часу та умов вимірювання.
- Історія використаних засобів пацієнтом: найменування, частота
застосування, тривалість, реакції (суб’єктивні/об’єктивні).
- Реакції шкіри та зворотній зв’язок — відгуки користувачів про
відчуття, проблеми або покращення після використання засобів.
- Висновки лікарів: призначення, оцінки ефективності лікування,
рекомендації.
- Статистичні метадані: частота візитів, тривалість лікування,
змінність показників шкіри.
Крім того, уся інформація буде анонімізована для навчання моделей
3. Навчання / Використання моделей
На перших етапах буде використано готові API рішень (наприклад,
OpenAI, Google Vertex AI, Amazon SageMaker), щоб швидко протестувати
гіпотези та створити базову систему рекомендацій.
У майбутньому - власне навчання та донавчання моделей на зібраних
даних для глибшої персоналізації, вищої точності та незалежності від
сторонніх сервісів.
Такий підхід передбачає використання нейронних мереж (MLP,
CNN) для класифікації типу шкіри та визначення ефективності засобів, а
також рекурентних моделей - для роботи з часовими рядами й
прогнозування змін стану шкіри.
3.3 Limitations and Exclusions
У межах першої версії релізу, а також майбутніх оновлень із
впровадженням ШІ, визначено низку обмежень та свідомо виключених
функцій, щоб зберегти фокус на безпеці, надійності та клінічній
коректності використання.
1. Неавтоматизоване призначення засобів догляду
Обмеження: Рішення про застосування конкретного засобу залишається
виключно за лікарем.
Причина виключення: Автоматичне призначення може не враховувати
клінічний контекст, алергії, супутні захворювання та інші індивідуальні
особливості. Навіть із впровадженням ШІ модель буде виконувати роль
рекомендаційної системи, а не замінювати лікаря.
2. Відсутність соціальної взаємодії між користувачами
Обмеження: Пацієнти не можуть публічно обговорювати засоби, ділитися
рекомендаціями чи створювати спільноти в межах додатку.
Причина виключення: Така взаємодія може створити дезінформацію,
впливати на сприйняття результатів без медичного підтвердження, а також
створити ризики для конфіденційності медичних даних.
3. Не передбачається використання з несертифікованими засобами
Обмеження: База даних засобів включатиме лише ті, що пройшли
перевірку чи сертифікацію.
Причина виключення: Це критично важливо для безпеки пацієнтів і
відповідає медичним вимогам — система не може підтримувати або
рекомендувати засоби з недоведеною ефективністю чи небезпекою
побічних ефектів.
4. Відсутність офлайн-режиму роботи з пристроєм
Обмеження: IoT-пристрій та мобільний додаток вимагають постійного
підключення до мережі для передачі та обробки даних.
Причина виключення: Без підключення неможливий обмін інформацією
між клієнтом і сервером, що унеможливлює коректне функціонування всієї
екосистеми в реальному часі.
5. Використання лише централізованої моделі ШІ
Обмеження: Модель ШІ буде навчатися та розгортатися лише на стороні
сервера.
Причина виключення: Розподілене навчання на пристроях (наприклад,
Federated Learning) наразі не розглядається через складність реалізації,
обмеження обчислювальних ресурсів і складності валідації медичних
даних.
